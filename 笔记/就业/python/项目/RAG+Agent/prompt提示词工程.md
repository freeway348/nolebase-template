# prompt提示词工程
### 一、提示词技巧

1. 详细的描述：将你想要得到的结果使用详尽的语言描述，越详细越具体，大模型生成的答案越接近你想要的结果
2. 让大模型充当某个角色：给大模型预设一个角色，让它作为该角色来执行你的要求
3. 使用分隔符标明输入的不同部分：可以使用中括号、XML标签、三引号等分隔符划分重要的文本，然后再指明分隔符内的内容是需要按照某些规定、要求进行分析、实现的
4. 对任务指定步骤：将任务拆分为多个详细步骤，让大模型按步骤来生成内容
5. 提供例子：先把例子给大模型，然后让大模型根据例子生成输出内容
	- 该技巧本质类似于`few-shot learning`。

### 二、提示词实战案例

#### 1. 背景

当前金融领域信息化发展的时代，金融数据大量激增，许多投资者和研究者试图通过对这些数据进行深度分析而获得一些有效的决策和帮助，尽可能减少决策失误带来的损失。

所以，针对金融数据的分析方法研究是目前十分有益且热门的话题。

当前案例主要有三大业务场景实现： 
- 基于大模型完成：金融文本分类
- 基于大模型完成：金融文本信息抽取
- 基于大模型完成：金融文本匹配

大模型选择：Qwen在线大模型

采用方法：基于`Few-Shot + Zero-Shot`的思想，设计提示词（prompt），进而应用大模型完成相应的任务

##### 1）Few-Shot


通过给予少样本示例，引导大模型对齐示例输出结果

##### 2）Zero-Shot

零样本，无提示，仅用语言描述任务，依赖模型的预训练知识回答
Zero-Shot是大模型通过**预训练**知识储备和强**泛化**能力的集中体现
